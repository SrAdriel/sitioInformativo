¿Cuál es el contexto en el que se usa el sistema?

El sistema ADUP es un programa que está orientado a agilizar las tareas de recolección de datos por parte de las comisarías y/o otros departamentos policiales, aportando en la agilización de los procesos de administración sobre información, creándose en base a la necesidad que hay en algunos centros policiales donde no se cuenta con una forma de registro de datos óptima, sencilla y por sobre todo segura, donde la información pueda estar resguardada, existiendo entonces este programa como solución a todos esos problemas.

¿Por qué un sistema para el uso policial?

El sistema policial es una parte esencial del aparato estatal, diseñado para mantener la seguridad pública, hacer cumplir la ley y proteger a los ciudadanos. Estos sistemas son redes complejas de agencias y fuerzas responsables de mantener el orden social y la paz, e investigar, prevenir y responder al crimen y las amenazas a la seguridad. El sistema policial opera en múltiples niveles de gobierno y en la mayoría de los países hay tres niveles principales: policía local, policía estatal y policía nacional o federal. Cada nivel gobierna un área específica, y su alcance geográfico y autoridad varían según el gobierno y la estructura legal de un país en particular.

Programas utilizados para el desarrollo del Sistema ADUP

Visual Studio

Visual Studio es un programa en el que se pueden crear sistemas desde cero con distintos lenguajes de programación, en este caso, se usó C# (C sharp) para la codificación del Sistema ADUP y la atribuciön de todas sus funciones gracias a sus herramientas de diseño.

SQL Server

Sql Server es un programa para la creación de bases de datos, es decir, su importancia para la Creación del Sistema ADUP es que todos los datos que se ingresen en este programa, irán a una base de datos que fue creada y configurada en SQL Server.
